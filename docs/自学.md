## 泛型
T,E,K,V表示任意数据类型，等待装填，`<T>`表示在编译阶段检查并保证输出T类型的数据，只能是引用类型，不能是基本数据类型  
如List`<String>`,就是说这个类是列表类，但是泛型可以保证里面的成员都是String类型，否则编译不通过，如ResultUtil`<T>`这种，也是可以用泛型来规定数据的类型  
## 迭代器
场景二：遍历的时候删东西，因为普通for和加强for删的时候不管位置变动，迭代器可以优雅的边遍历边删除元素
  // 错误写法
  List`<String>` list = new ArrayList<>();
  list.add("A");
  list.add("B");
  list.add("C");
  for (String s : list) {
      if (s.equals("B")) {
          list.remove(s);  // 炸！ConcurrentModificationException
      }
  }
  // 正确写法：用迭代器
  Iterator`<String>` it = list.iterator();
  while (it.hasNext()) {
      if (it.next().equals("B")) {
          it.remove();  // 安全删除
      }
  }
