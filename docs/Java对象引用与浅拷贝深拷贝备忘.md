# Java：对象引用、`roomMap` 与浅拷贝 / 深拷贝（备忘）

面向：**已经会写 Java，但对「变量里到底存了什么」容易想当然** 的同学。  
本仓库里 **`DpRoomServiceImpl`** 从 **`roomMap`** 取房间、再传给各种方法改状态，以及 **`deepCopyRoomForSnapshot`** 用 Jackson 做快照，都和下面这套概念直接相关。

---

## 1. 先分清：`int` 和「房间对象」不是一回事

```java
int a = 1;
int b = a;
a = 99;
// b 还是 1
```

这里变量 **`a` / `b` 的小格子里装的就是数字本身**。给 `b = a` 等于把数字 **再抄一份**，改 `a` 不会动到 `b`。

---

## 2. 对象变量：格子里往往是「门牌号」，不是整间屋子

像 **`DpRoom`**、**`List`**、**`Map`**、数组这类（除了 Java 的 **8 种基本类型** 以外的大部分东西），变量里通常存的是 **引用**——可以理解为堆里那个对象的 **地址 / 门牌号**。

所以：

```text
roomMap 里 roomId → 指向内存里唯一的那间「房间对象」

DpRoom room = roomMap.get(roomId);   // room 是又一个名字，指向同一间屋
someMethod(room);                    // 传进去的也是同一把「钥匙」
// 在方法里改 room 的字段、改 room.getPlayers() 那个列表……
// 改的都是 map 里那一份，不是「又复制了一套房」
```

**误区**：以为 `DpRoom room = roomMap.get(...)` 会拿到一个 **新的** 房间。  
**实际**：只是多一个引用名；**除非**代码里显式 `new DpRoom()` 或做了 **拷贝**，否则大家改的都是 **同一对象**。

---

## 3. 除了「类的实例」，还有哪些通常是引用？

| 常见类型 | 是不是引用（共享同一块） |
|----------|--------------------------|
| `int`、`long`、`double`、`boolean`、`char` 等基本类型 | 否，赋值一般是 **值的复制** |
| 任意 `class` / `interface` 实例 | 是 |
| **数组**（包括 `int[]`：数组对象是引用；元素里的 `int` 仍是值） | 数组本身是引用 |
| **`List` / `Map` / `Set`** | 是 |
| **`String`** | 是引用，但 **`String` 不可变**，看起来不会像可变列表那样「改内容」 |

读代码时粗规则：**类型不是基本类型，就按「可能是同一块内存」来想**；看见 **`list.add`、`map.put`、setter 改字段**，就要想到 **可能所有指着它的地方都会变**。

---

## 4. 浅拷贝 vs 深拷贝（人话版）

- **浅拷贝**：可能 `new` 了一个 **外层** 对象，但里面的 **`List`、`Map`、嵌套的玩家对象** 仍和原来 **共用**——像换了个新文件夹封面，纸条还是同一张。
- **深拷贝**：**里面嵌套的** 列表、映射、子对象也各自 **再复制一份**，改副本 **不会** 动到原件。

**为什么**「新建一个 `DpRoom` 再 `setPlayers(live.getPlayers())`」往往不够？  
因为 **`setPlayers` 传进去的仍是同一个 `List` 引用**，改列表内容，**原件也跟着变**——这是浅拷贝的典型坑。

本仓库里 **`deepCopyRoomForSnapshot`** 用 **`ObjectMapper` 先序列化再反序列化**，是在 **可序列化字段** 上整棵树复制一份，避免手写每层 `new` 时漏字段；代价是多一点 CPU / 内存，换来 **快照与运行中房间互不干扰**。

## 5. 和本项目的对应关系（方便搜代码）

| 你在想什么 | 代码里可能在干什么 |
|------------|-------------------|
| 「我只是在改局部变量 `room`」 | `room` 和 `roomMap` 里是 **同一对象**，改字段 = 改全局那份房间状态 |
| 「传给方法的是副本」 | 传的是 **引用的值**，方法里改对象内容，**外面也看得见** |
| 「我要某一瞬间的房间给别人看，且不能影响后面逻辑」 | 需要 **深拷贝** 或专门只暴露 DTO，而不是把 `DpRoom` 引用直接交出去乱改 |

相关实现线索：`DpRoomServiceImpl#deepCopyRoomForSnapshot`、`DpRoom` 内大量 `List` / `Map` 字段。

## 6. 一句话收工

- **`int b = a`**：抄的是 **数字**。  
- **`Room r = map.get(id)`**：多的是 **指向同一间房的钥匙**；要「另一间房」得 **`new` + 真拷贝** 或项目里的快照方案。

学很久才反应过来很正常：**课本爱讲语法，少强调「引用」在业务代码里天天踩坑。** 以后看到 **`get` 出来一路传下去改**，先问一句：**我们是在同一间房里搬家具吗？**
--- 
  
      private DpRoom deepCopyRoomForSnapshot(DpRoom live) {
        try {
            return objectMapper.readValue(objectMapper.writeValueAsBytes(live), DpRoom.class);
        } catch (Exception e) {
            throw new IllegalStateException("room snapshot copy failed", e);
        }
    }  
    这才是深拷贝，真正的复制一份全新的实例
